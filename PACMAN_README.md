# 🎮 吃豆人颈椎大冒险 - 游戏化颈椎呵护助手

## 🎯 设计理念

将传统的颈椎提醒应用改造为经典的**吃豆人 (Pac-Man)** 游戏，通过人脸移动控制游戏角色，让颈椎锻炼变得有趣！

## 🕹️ 游戏控制

### 人脸移动映射

| 头部动作 | 吃豆人移动 | 颈椎锻炼效果 |
|---------|-----------|-------------|
| **向左看** | 向左移动 | 颈部左侧拉伸 |
| **向右看** | 向右移动 | 颈部右侧拉伸 |
| **向上看** | 向上移动 | 颈部后侧伸展 |
| **向下看** | 向下移动 | 颈部前侧伸展 |

### 控制灵敏度

- **阈值**: 头部转动超过 10° 才触发移动
- **速度**: 吃豆人移动速度与头部转动幅度成正比
- **平滑**: 移动采用平滑过渡，避免跳跃

## 🎮 游戏机制

### 目标
- 吃掉所有豆子获得分数
- 躲避四个幽灵的追击
- 收集能量道具获得特殊能力

### 得分系统
- 🍬 **普通豆子**: 10 分
- ⚡ **能量道具**: 50 分
- 👻 **冻结幽灵时**: 额外奖励

### 能量道具
1. **🏃 速度提升** - 吃豆人移动速度加倍（10 秒）
2. **👻 幽灵冻结** - 幽灵变蓝无法追击（10 秒）
3. **✨ 双倍分数** - 所有得分翻倍（10 秒）

### 关卡系统
- 每吃完所有豆子进入下一关
- 每关幽灵速度递增
- 无限关卡，挑战高分

## ⏰ 健康提醒

### 自动休息
- 每 **5 分钟** 游戏时间触发休息提醒
- 休息倒计时 **30 秒**
- 强制进行颈椎放松锻炼

### 锻炼统计
- 🔄 **活动次数**: 记录头部移动次数
- ⏱ **游戏时间**: 累计游戏时长
- 🍬 **得分**: 游戏分数统计

## 📊 技术实现

### 核心技术
- **Face-API.js**: 人脸检测与特征点识别
- **Canvas API**: 游戏渲染
- **MediaDevices API**: 摄像头访问

### 头部姿态计算
```javascript
// Yaw (左右旋转) - 通过鼻尖与鼻根的水平偏移计算
yaw = (noseTip.x - noseBase.x) / 8

// Pitch (上下倾斜) - 通过鼻尖与鼻根的垂直偏移计算
pitch = (noseTip.y - noseBase.y) / 8

// Roll (侧倾) - 通过双眼连线角度计算
roll = atan2(rightEye.y - leftEye.y, rightEye.x - leftEye.x)
```

## 💡 健康建议

### 正确姿势
1. 将设备放在与眼睛平齐的位置
2. 保持 30-50cm 的观看距离
3. 确保光线充足，避免反光

### 锻炼建议
- 每局游戏后做颈部拉伸
- 感到不适立即停止
- 配合肩部和背部放松

### 注意事项
⚠️ **不要**:
- 长时间保持同一姿势
- 过度快速转动头部
- 在移动的车辆上使用

✅ **建议**:
- 每 30 分钟起身活动
- 配合深呼吸
- 多喝水保持水分

## 🔧 自定义配置

在代码中修改 `CONFIG` 对象:

```javascript
const CONFIG = {
    gridSize: 20,          // 游戏网格大小
    pacmanSpeed: 1,        // 吃豆人基础速度
    ghostSpeed: 0.5,       // 幽灵基础速度
    warningTime: 5 * 60 * 1000,  // 休息提醒时间 (毫秒)
    exerciseTime: 30       // 锻炼倒计时 (秒)
};
```

### 调整灵敏度
修改 `controlPacman` 函数中的 `threshold` 值:
- **降低** (如 5): 更灵敏，轻微转动即可控制
- **提高** (如 15): 需要更大转动幅度

## 🐛 故障排除

### 摄像头无法启动
- 检查浏览器权限设置
- 确保没有其他应用占用摄像头
- 尝试刷新页面

### 人脸检测不稳定
- 确保光线充足
- 面部完整出现在画面中
- 避免快速大幅度移动

### 游戏卡顿
- 关闭其他浏览器标签页
- 降低视频分辨率
- 检查设备性能
